## Usage Examples

### Basic Translation
```typescript
import { useLanguage } from '@/hooks/useLanguage';

const MyComponent = () => {
  const { t } = useLanguage();
  
  return <h1>{t('products:shopByCategory')}</h1>;
};
```

### Translation with Variables
```typescript
const { t } = useLanguage();

return (
  <p>
    {t('products:weFound')} <strong>{count}</strong> {t('products:itemsForYou')}
  </p>
);
```

### Nested Translation Keys
```typescript
const { t } = useLanguage();

return (
  <div>
    <h3>{t('home:features.freeShipping.title')}</h3>
    <p>{t('home:features.freeShipping.description')}</p>
  </div>
);
```

## Adding New Translations

### Step 1: Choose the Right File
- **Common UI elements** → `common.json`
- **Navigation** → `navigation.json`
- **Product-related** → `products.json`
- **Page-specific** → Create or use page-specific file

### Step 2: Add to Both Languages
```json
// en/products.json
{
  "newFeature": "New Feature Text"
}

// ar/products.json
{
  "newFeature": "نص الميزة الجديدة"
}
```

### Step 3: Use in Component
```typescript
{t('products:newFeature')}
```

## Translation Best Practices

1. **Use Namespaces** - Always prefix with namespace (e.g., `products:`, `common:`)
2. **Descriptive Keys** - Use clear, descriptive key names
3. **Consistent Structure** - Group related translations together
4. **No Hardcoded Text** - Always use translation keys
5. **Test Both Languages** - Verify translations in English and Arabic
6. **Handle Plurals** - Use separate keys for singular/plural when needed
7. **Keep It Simple** - Break complex sentences into smaller parts

<p>{t('products:weFound')} {count} {t('products:itemsForYou')}</p>

### Pattern 3: Conditional Text
```typescript
{isLoading ? t('common:messages.loading') : t('common:messages.success')}
```

### Pattern 4: Dropdown Options
```typescript
<option value="featured">
  {t('products:sorting.sortBy')}: {t('products:sorting.featured')}
</option>
```

## Troubleshooting

### Translation Not Showing
1. Check translation key exists in both `en` and `ar` files
2. Verify namespace is correct (e.g., `products:`, not `product:`)
3. Ensure translation files are loaded (check `src/i18n/config.ts`)
4. Check browser console for i18next errors

### Wrong Language Displaying
1. Check language switcher is working
2. Verify localStorage has correct language preference
3. Check `document.documentElement.lang` attribute
4. Ensure `useLanguage` hook is being used

### RTL Layout Issues
1. Verify `document.documentElement.dir` is set to `rtl`
2. Check tailwindcss-rtl plugin is installed
3. Use logical CSS properties (margin-inline-start vs margin-left)
4. Test with Arabic language selected
